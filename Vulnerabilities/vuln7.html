Identification and Authentication Failures - Vulnerability 7

Description of Identification and Authentication Failures

Key terms related to this vulnerability:

1. Identification: The process of uniquely recognizing a user before authentication.

2. Authentication: The method used to verify the identity of a user, such as passwords, biometrics, or OTPs.

3. Multi-Factor Authentication (MFA): An authentication method that requires multiple verification factors to enhance security.

4. Session Management: The mechanism used to maintain a user's authenticated session securely.

Identification and Authentication Failures: This vulnerability occurs when authentication mechanisms are weak, allowing attackers to bypass authentication and gain unauthorized access. Common Risks:

  • Weak Password Policies: Allowing short or predictable passwords.
  • Exposed Credentials: Storing passwords in plaintext or sending them over insecure connections.
  • Brute Force Attacks: Attackers guessing passwords through automated tools.
  • Session Fixation: Reusing session IDs to hijack authenticated sessions.

Countermeasures:
  1. Implement strong password policies and enforce complexity rules.
  2. Use Multi-Factor Authentication (MFA) to strengthen authentication.
  3. Secure session management by using short-lived, randomly generated session IDs.
  4. Encrypt sensitive authentication data in transit and at rest.
  5. Limit failed login attempts and implement CAPTCHA for bots.

Example of both vulnerable and secure authentication implementations:
  • Vulnerable Authentication: The application allows weak passwords, stores them in plaintext, and does not limit failed login attempts.
  • Secure Authentication: The application enforces strong passwords, securely hashes passwords, and implements account lockout on multiple failed attempts.

Vulnerable Authentication


# Allows weak passwords and stores them in plaintext (Vulnerable)
def register(username, password):
    with open("users.txt", "a") as file:
        file.write(f"{username}:{password}\n")  # Storing passwords in plaintext
    return "User registered."

Secure Authentication


import bcrypt

# Enforces strong passwords and hashes them securely (Secure)
def register(username, password):
    hashed_pw = bcrypt.hashpw(password.encode(), bcrypt.gensalt())
    with open("users.txt", "a") as file:
        file.write(f"{username}:{hashed_pw.decode()}\n")  # Storing hashed passwords
    return "User registered securely."
